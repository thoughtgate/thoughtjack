# Add a Scenario

Attack scenarios are authored as OATF (Open Agentic Testing Framework) documents. The official scenario library and authoring guide live on [oatf.dev](https://oatf.dev).

## Writing a scenario

OATF documents are declarative YAML files that describe multi-phase adversarial tests. See the [OATF scenario authoring guide](https://oatf.dev/docs/authoring) for the full specification, including:

- Document structure (actors, phases, triggers, responses)
- Extractor and interpolation syntax
- Indicator definitions and verdict criteria
- Security framework mappings (MITRE ATT&CK, OWASP MCP Top 10)

## Running a scenario with ThoughtJack

Once you have an OATF document, run it with ThoughtJack:

```bash
# Validate the document
thoughtjack validate scenarios/my_attack.yaml

# Execute the scenario
thoughtjack run --config scenarios/my_attack.yaml -vv
```

## Publishing to the OATF library

To contribute scenarios to the official library, follow the [OATF contribution guide](https://oatf.dev/docs/contributing). Published scenarios are available to all OATF-compatible tools, including ThoughtJack.
