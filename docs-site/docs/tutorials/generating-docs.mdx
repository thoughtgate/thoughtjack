# Generating Documentation

In this tutorial you'll build ThoughtJack's attack scenario catalog site. The `docgen` module reads built-in scenarios and produces Docusaurus-compatible MDX pages with metadata cards, Mermaid diagrams, and YAML source toggles.

:::info
The `docgen` module exists in source (`src/docgen/`) but is not yet exposed as a CLI command. Documentation generation is done as part of the docs site build process.
:::

## 1. Understand what gets generated

Each scenario page includes:

- **Frontmatter** — title, sidebar position, description for SEO
- **Attack metadata card** — severity, MITRE ATT&CK IDs, OWASP MCP Top 10 mapping, tags
- **Description** — what the scenario does and what to observe
- **Phase diagram** — Mermaid state diagram showing phase transitions
- **YAML source** — toggleable full scenario configuration

The generator also creates:
- `sidebars.js` — sidebar navigation grouped by category
- Coverage pages linking scenarios to security framework mappings

## 2. Build the docs site

The documentation site uses Docusaurus. Install dependencies and build:

```bash
cd docs-site
npm install
npm run build
```

The generated site includes:
- **Documentation** — tutorials, how-to guides, reference, and explanation pages
- **Attack Catalog** — scenario pages
- **Coverage** — MITRE ATT&CK and OWASP MCP Top 10 matrices

## 3. Preview locally

```bash
cd docs-site
npm start
```

This starts a local development server at `http://localhost:3000/thoughtjack/`. Navigate to the Attack Catalog section to see the scenario pages.

## How static and generated pages coexist

The docs site has two sidebar configurations:

- **`docs`** — static Diataxis pages (tutorials, how-to, reference, explanation) defined in `sidebars.ts`
- **`scenarios`** — scenario sidebar from `sidebars.js`

The `sidebars.ts` file conditionally loads the `sidebars.js`. If the scenarios sidebar hasn't been generated yet, the site builds without the scenario catalog — the static documentation pages still work.
